# Laudato Si' 3D Mission

สื่อออนไลน์เกมการศึกษา 3 มิติ เรื่อง **Laudato Si' 3D Mission: ภารกิจพิทักษ์บ้านส่วนรวม** สำหรับนักเรียนชั้น ม.6 ใช้เล่นรายบุคคลบนโทรศัพท์หรือคอมพิวเตอร์ได้จากไฟล์เว็บเดียว

## ไฟล์หลัก

- `index.html` - หน้าเกมหลัก มีแผนที่โรงเรียน 3 มิติ ภารกิจ คะแนน เหรียญ สมุดภารกิจ และแผงครูดูผล
- `server.js` - local web server และ API ฐานข้อมูลผลคะแนนแบบไฟล์
- `teacher-results.html` - หน้าครูสำหรับดูผลคะแนนที่บันทึกในฐานข้อมูล local
- `Code.gs` - โค้ด Google Apps Script สำหรับรับคะแนนจากเกมและบันทึกลง Google Sheet
- `.github/workflows/ci.yml` - GitHub Actions สำหรับตรวจโปรเจกต์อัตโนมัติ
- `GITHUB_SETUP.md` - คู่มือ clone, run, test, commit และ push ขึ้น GitHub
- `innovation-laudato-si-challenge.md` - เอกสารอธิบายนวัตกรรมการจัดการเรียนรู้
- `registration-laudato-si-challenge.md` - แนวทางเชื่อมระบบลงทะเบียน/ผลคะแนนกับ Google Sheet
- `design-concept.png` - ภาพแนวคิดเดิมที่ใช้เป็นทิศทางภาพรวมของสื่อ

## จุดเด่นของเกม

- นักเรียนรับบทเป็น “ผู้พิทักษ์บ้านส่วนรวม”
- สำรวจโรงเรียน 3 มิติด้วยปุ่มทิศทางหรือคีย์บอร์ด
- เล่น 6 ภารกิจตามลำดับ พร้อมปลดล็อกเหรียญรายบุคคล
- ทุกภารกิจมีเกมตอบคำถามแบบเลือกตอบ พร้อม feedback ถูก/ผิด และคิดคะแนนร่วมกับคำตอบเขียน
- คะแนนเต็ม 250 คะแนน รวมคะแนนภารกิจ 230 คะแนน ความรับผิดชอบ 10 คะแนน และความคิดสร้างสรรค์พิเศษ 10 คะแนน
- มีสมุดภารกิจสำหรับดูคำตอบ คะแนน เหรียญ และคำมั่นสัญญา
- ครูสามารถดูคะแนนรายบุคคล ดาวน์โหลดรายงาน และส่งข้อมูลไป Google Sheet ได้

## การรันเว็บและฐานข้อมูลพร้อมใช้งาน

รันคำสั่งนี้จากโฟลเดอร์โปรเจกต์:

```powershell
npm start
```

เมื่อรันสำเร็จให้เปิด:

- หน้าเกม: `http://localhost:3000`
- หน้าฐานข้อมูลครู: `http://localhost:3000/teacher-results.html`
- Health check: `http://localhost:3000/api/health`

เมื่อนักเรียนกด “ส่งคะแนนไปฐานข้อมูล” ระบบจะบันทึกข้อมูลไว้ที่:

- `data/submissions.jsonl`
- `data/submissions.csv`

ถ้า port `3000` ถูกใช้งานอยู่ server จะเลื่อนไปใช้ port ถัดไปโดยอัตโนมัติและแสดง URL ใน terminal

## การตรวจสำหรับ GitHub

ก่อน push ขึ้น GitHub ให้รัน:

```powershell
npm test
```

GitHub Actions จะรันคำสั่งเดียวกันอัตโนมัติเมื่อ push หรือเปิด pull request เข้า `main`/`master`

## การใช้งานแบบไฟล์ในเครื่อง

เปิด `index.html` ด้วยเว็บเบราว์เซอร์ได้ทันที ไม่ต้องติดตั้งโปรแกรมเพิ่ม ข้อมูลจะบันทึกในเครื่องผู้เล่นผ่าน `localStorage`

## การใช้งานกับ Google Sheet

ลิงก์ชีตที่ตั้งค่าไว้ในโค้ด:

`https://docs.google.com/spreadsheets/d/1QhY6qeVED7hSs99vVfg-ABin3Az9RWDW6ua3EJbcj58/edit?usp=sharing`

ขั้นตอน:

1. เปิด Apps Script project ที่ผู้ใช้ส่งมา
2. วางโค้ดจาก `Code.gs` ลงในไฟล์ `Code.gs`
3. สร้างไฟล์ HTML ชื่อ `Index` แล้ววางเนื้อหาจาก `index.html`
4. กด Run ฟังก์ชัน `setupSheet` หนึ่งครั้งเพื่อสร้างหัวตาราง
5. Deploy เป็น Web App โดยเลือก Execute as: Me และกำหนดสิทธิ์ผู้เข้าใช้ตามบริบทของโรงเรียน
6. เปิด Web App URL ที่ได้ นักเรียนจะเล่นเกมและส่งผลคะแนนเข้าชีตได้จากปุ่ม “ส่งคะแนนไป Google Sheet”

ถ้ายังเปิดจากไฟล์ `index.html` ในเครื่อง ปุ่มส่งคะแนนจะยังไม่ส่งจริงจนกว่าจะนำ Web App URL ที่ deploy แล้วไปใส่ในตัวแปร `APPS_SCRIPT_WEB_APP_URL`
